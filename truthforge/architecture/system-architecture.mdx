## High-Level Architecture

TruthForge is built as a **layered system**. A user request flows from the browser, through a REST API, into an orchestrator that coordinates 5 specialized AI agents, and ends with a tamper-proof record on the Hedera blockchain.

## Request Lifecycle

This is what happens from the moment a user submits a verification request to when they receive a result.

```mermaid
sequenceDiagram
    participant U as User / Frontend
    participant API as Flask API
    participant OA as Orchestrator Agent
    participant VA as Verification Agent
    participant CA as Carrier Agent
    participant EA as Evidence Agent
    participant HCS as Hedera HCS
    participant DB as PostgreSQL

    U->>API: POST /api/verify (image + tracking number)
    API->>OA: process_request()
    OA->>OA: parse_intent() → route_request()

    par Parallel Agent Execution
        OA->>VA: analyze_image() + verify_document()
        VA->>VA: EXIF → Lighting → AI Artifacts → Metadata
        VA-->>OA: authenticity_score + compliance_report
    and
        OA->>CA: query_shipment(tracking_number)
        CA->>CA: normalize_tracking_data()
        CA-->>OA: unified_shipment_data
    end

    OA->>EA: create_audit_trail(results)
    EA->>HCS: submit_consensus() → transaction_id
    HCS-->>EA: consensus_timestamp
    EA-->>OA: audit_reference + hcs_tx_id

    OA->>DB: store verification + audit trail
    OA->>OA: aggregate_results() → UnifiedReport
    OA-->>API: UnifiedReport
    API-->>U: JSON response with score + HCS proof
```

## Component Breakdown

### The 5-Agent Network

Each agent is independently registered on Hedera's HOL (Hashgraph Online) protocol, making them discoverable and independently scalable.

```mermaid
graph LR
    subgraph "Agent Capabilities"
        OA["🧠 Orchestrator<br/>orch-001<br/>─────────────<br/>• Parse natural language<br/>• Route requests<br/>• Aggregate results<br/>• Coordinate workflows"]
        VA["🔍 Verification<br/>verify-001<br/>─────────────<br/>• EXIF analysis<br/>• Lighting detection<br/>• AI artifact scan<br/>• BOL validation"]
        CA["🚚 Carrier Adapter<br/>carrier-001<br/>─────────────<br/>• FedEx integration<br/>• UPS / DHL support<br/>• Data normalization<br/>• Rate limit handling"]
        RA["📡 Registry<br/>registry-001<br/>─────────────<br/>• Agent discovery<br/>• Health monitoring<br/>• HOL sync<br/>• TTL caching"]
        EA["📜 Evidence<br/>evidence-001<br/>─────────────<br/>• HCS submission<br/>• Audit references<br/>• HBAR cost tracking<br/>• Retry logic"]
    end

    OA --> VA & CA & RA & EA
```

### 4-Layer Deepfake Detection

The Verification Agent runs every cargo photo through 4 independent analysis layers. Each layer produces a confidence score; they are combined into a final **Authenticity Score (0–100)**.

```mermaid
flowchart LR
    IMG[📷 Cargo Photo] --> L1

    subgraph "4-Layer Pipeline"
        L1["Layer 1: EXIF Analysis<br/>Camera make/model<br/>GPS coordinates<br/>Capture timestamp<br/>Tampering indicators"]
        L2["Layer 2: Lighting Analysis<br/>Shadow consistency<br/>Color temperature<br/>Reflection patterns<br/>Anomaly detection"]
        L3["Layer 3: AI Artifact Detection<br/>Unnatural edges<br/>Blurring patterns<br/>Generation probability<br/>Bounding box mapping"]
        L4["Layer 4: Metadata Verification<br/>File format integrity<br/>Compression artifacts<br/>Edit history<br/>Dimension consistency"]
    end

    L1 --> SCORE
    L2 --> SCORE
    L3 --> SCORE
    L4 --> SCORE

    SCORE["🎯 Authenticity Score<br/>0 – 100<br/>+ Compliance Report"]
    SCORE --> HCS["⛓️ HCS Timestamp"]

    L1 --> L2 --> L3 --> L4
```

### HCS-10 Messaging Protocol

All agent-to-agent communication uses the HCS-10 standard — every message is submitted to a Hedera topic, creating an immutable audit trail.

```mermaid
graph TD
    MSG["HCS10Message<br/>─────────────────<br/>message_type: REQUEST/RESPONSE/<br/>QUERY/NOTIFY/DISCOVER<br/>sender_id: agent ID<br/>recipient_id: agent ID<br/>timestamp: ISO 8601<br/>payload: JSON dict<br/>signature: cryptographic hash"]

    MSG -->|submit_message| TOPIC["Hedera HCS Topic<br/>0.0.8161249"]
    TOPIC -->|returns| TXN["Transaction ID<br/>e.g. 0.0.7974354@..."]
    TXN -->|stored in| DB["PostgreSQL<br/>audit_trails table"]
    TXN -->|viewable on| HS["HashScan Explorer"]
```

## Mock vs Live Mode

TruthForge supports two operating modes, switchable via a single environment variable.

```mermaid
flowchart TD
    ENV{MOCK_MODE=?}

    ENV -->|true| MOCK["🧪 Mock Mode<br/>─────────────────────<br/>• Fake HCS transaction IDs<br/>• Simulated carrier data<br/>• No HBAR costs<br/>• SQLite in-memory DB<br/>• Instant responses<br/>• Safe for development"]

    ENV -->|false| LIVE["🔴 Live Mode<br/>─────────────────────<br/>• Real Hedera Testnet TXNs<br/>• Real FedEx API calls<br/>• HBAR balance required<br/>• PostgreSQL (Supabase)<br/>• Transaction receipts waited<br/>• Exponential backoff retries"]
```
